Introduction to Composer

David Mosher / @arenowebdev

What Is It?

  • It was inspired by Ruby's Bundler and Node's npm

  • It is a tool for dependency management NOT a package manager

  • It is cross-platform compatible (requires >= PHP 5.3.2)
  • Though it does deal with "packages" or libraries, it manages them on a per-project basis. Hence, it is a dependency manager

Why Use It?

  • Your project has dependencies

  • Those dependencies also have dependencies

  • Unzipping and manually moving files sucks

  • Updating those unzipped and moved files sucks more

  • Including each of those dependencies sucks the most

You specify what your project needs in writing

Installing It

Locally (only within your project)
$ curl -sS http://getcomposer.org/installer | php

Globally (system wide - recommended)
$ curl -sS http://getcomposer.org/installer | php
$ sudo mv composer.phar /usr/local/bin/composer
  • I'm only going to cover installing it on non-windows machines.

Virtual Packages

  • php
    Allows you to apply constraints on the PHP version
    "php": ">=5.3.2"

  • extname
    Allows requiring of PHP extensions
    "extcurl": "*"

  • lib<name>
    Allows version constraints of PHP libraries
    "libpcre": ">=7.8"

$ composer install

Fruits of That Labor

Autoloading

"Besides downloading the libraries outlined in your composer.json file, Composer also prepares an autoload file that's capable of autoloading all of those classes in any of the libraries it downloaded."

To use your dependencies in your project, simply require the autoload file as such:


require 'vendor/autoload.php';

You now have full access to any of the namespaced classes!

You can include other files in this autoloader. There are three methods of doing so:

PSR-0

"autoload": {
    "psr0": { "Mosher\\Presentation\\": "src/" }
}

Classmap

"autoload": {
    "classmap": [ "lib/", "ext/library.php" ]
}

Files

"autoload": {
    "files": [ "ext/functions.php" ]
}
